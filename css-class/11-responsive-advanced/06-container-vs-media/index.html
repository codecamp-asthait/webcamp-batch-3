Container Queries vs Media Queries

The Evolution of Responsive Design

Media queries respond to viewport size. Container queries respond to component size. They solve different problems.

Media Queries: Viewport-Based

Traditional approach - responds to browser window size:

Media Query Example

@media (min-width: 768px) {
    .card {
        display: grid;
        grid-template-columns: 1fr 1fr;
    }
}

Responsive Card

This card uses media queries to respond to viewport width.

Container Queries: Size-Based

Modern approach - responds to parent container size:

Container Query Example

@container (min-width: 300px) {
    .card {
        display: grid;
        grid-template-columns: 1fr 1fr;
    }
}

When to Use Each

Media Queries

  • Page layout changes
  • Global design adjustments
  • Broad viewport adaptation
  • Browser support critical

Container Queries

  • Component styling
  • Reusable components
  • Modular design
  • Context independence

Container Query Syntax

Container Type

Define container context:

.container {
    container-type: inline-size;
}

Named Container

Query specific containers:

@container page (min-width: 800px) {
    .card { display: grid; }
}

Container Units

Size relative to container:

.item {
    width: 50cqw; /* 50% of width */
    font-size: 2cqmin; /* 2% min dim */
}

Progressive Enhancement

Fallback Pattern

/* Default - works everywhere */
.card { display: block; }

/* Enhanced - modern browsers */
@supports (container-type: inline-size) {
    .container {
        container-type: inline-size;
    }
    
    @container (min-width: 400px) {
        .card { display: grid; }
    }
}

Migration Strategy

For Existing Projects

  1. Keep media queries for pages
  2. Add container queries to components
  3. Test in supported browsers
  4. Provide fallbacks

For New Projects

  1. Use container queries first
  2. Use media queries for layout
  3. Plan progressive enhancement
  4. Document approach clearly

Best Practices

1. Use Media Queries for Layout

Page-level structural changes respond to viewport

2. Use Container Queries for Components

Reusable components respond to available space

3. Test Multiple Contexts

Verify components work in different layouts

4. Provide Fallbacks

Ensure graceful degradation in older browsers

The Future of Responsive Design

Container queries represent the next evolution in responsive web design. As browser support expands, they will become the primary way to build responsive components, while media queries remain for page layout and global changes.

Today

Container queries supported in modern browsers. Fallbacks needed.

Soon

Broader browser support. Container queries mainstream.

Future

Universal support. Component-first responsive design.
