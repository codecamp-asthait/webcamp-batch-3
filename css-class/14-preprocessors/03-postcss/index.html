PostCSS

What is PostCSS?

PostCSS is a JavaScript tool that transforms CSS using plugins. It parses CSS, applies transformations through a modular plugin system, and outputs the modified CSS.

Unlike preprocessors like Sass, PostCSS works with standard CSS and uses plugins for transformations.

How PostCSS Works

1

Parse

Input CSS is parsed into an AST (Abstract Syntax Tree)

2

Transform

Plugins analyze and modify the AST

3

Output

Modified CSS is generated from the AST

Popular Plugins

Autoprefixer

Automatically adds vendor prefixes for cross-browser compatibility

require('autoprefixer')

CSSnano

Minifies and optimizes CSS for production

require('cssnano')

PostCSS Preset Env

Converts modern CSS to compatible code

require('postcss-preset-env')

PostCSS Import

Inlines @import rules for bundling

require('postcss-import')

Autoprefixer Example

Before (Source CSS)

.grid {
    display: grid;
    gap: 20px;
}

.flex {
    display: flex;
    align-items: center;
}

After (Autoprefixed)

.grid {
    display: -ms-grid;
    display: grid;
    gap: 20px;
}

.flex {
    display: -webkit-box;
    display: -ms-flexbox;
    display: flex;
    -webkit-box-align: center;
    -ms-flex-align: center;
    align-items: center;
}

Configuration Example

postcss.config.js

module.exports = {
    plugins: [
        require('postcss-import'),
        require('postcss-preset-env')({
            stage: 3
        }),
        require('autoprefixer'),
        require('cssnano')({
            preset: ['default', {
                discardComments: {
                    removeAll: true
                }
            }]
        })
    ]
}

Installation & Setup

1. Install PostCSS

npm install -D postcss postcss-cli

2. Install Plugins

npm install -D autoprefixer cssnano

3. Create Configuration

touch postcss.config.js

4. Process CSS

postcss input.css -o output.css

PostCSS vs Preprocessors

PostCSS

  • Standard CSS syntax
  • Plugin-based
  • Modular approach
  • Transformations
  • No compilation step

Preprocessors (Sass/Less)

  • Custom syntax
  • All-in-one
  • Monolithic
  • Features built-in
  • Requires compilation
