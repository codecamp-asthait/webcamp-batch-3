Code Examples

Perspective with Transform Origin

.scene {
    perspective: 1000px;
    perspective-origin: 50% 50%;
}

.element {
    transform: rotateY(45deg);
}

/* Different vanishing points */
.top-left {
    perspective-origin: 0% 0%;
}

.bottom-right {
    perspective-origin: 100% 100%;
}

3D Card Stack

.card-stack {
    perspective: 1200px;
    transform-style: preserve-3d;
}

.card {
    position: absolute;
    transition: transform 0.3s;
    backface-visibility: hidden;
}

.card:nth-child(1) { transform: translateZ(0px); }
.card:nth-child(2) { transform: translateZ(-10px) rotateY(-2deg); }
.card:nth-child(3) { transform: translateZ(-20px) rotateY(-4deg); }
.card:nth-child(4) { transform: translateZ(-30px) rotateY(-6deg); }

.card-stack:hover .card:nth-child(1) {
    transform: rotateY(5deg) translateZ(20px);
}

Responsive Perspective

.scene {
    perspective: 1000px;
}

@media (max-width: 768px) {
    .scene {
        perspective: 600px;
    }
}

@media (prefers-reduced-motion: reduce) {
    .scene {
        perspective: none;
    }
}

Best Practices

  • Apply perspective to parent container, not individual elements
  • Use perspective values between 600px and 2000px
  • Combine with transform-style: preserve-3d for nested transforms
  • Set perspective-origin intentionally for specific effects
  • Test on actual devices for perspective rendering
  • Use CSS variables for consistent perspective values
  • Respect prefers-reduced-motion for accessibility
  • Add will-change: transform for animated 3D elements
